## Learned User Preferences

- Prefer auto-calculation of derived values over manual entry — if one number can be computed from others, compute it rather than asking the user to type it
- Design features for "full control": use flexible optional fields rather than rigid required ones (e.g. unit price is optional when every package has a fixed price)
- Bidirectional data entry: when A can derive B and B can derive A, support both directions so the user can start from whichever value they know
- When building an APK, copy the output file to the clipboard as a file (not just the path as text) so it can be pasted directly
- Prefer professional, theme-native UI — no gradients, rainbow accents, or decorative flourishes; use standard Material components, `ColorScheme` tokens, and flat/bordered panels

## Learned Workspace Facts

- Flutter POS/retail app using Riverpod for state, go_router for navigation, and JSON-based persistence via StorageHelper (not SQL); Firebase (Firestore + Auth) is used only for admin/licensing and cloud sync, never for local catalog/sales
- Product packages support per-package pricing (`packagePrice`), cost (`packageCostPrice`), inventory (`packageCount`), loose stock (`looseStock`), and bidirectional stock calc with `sellingPriceForPackage()` centralizing price resolution; `SaleItem.costPrice` captures per-base-unit cost at sale time (normalized from `packageCostPrice`) and profit displays on the catalog, receipts, receipt detail, and dashboard
- `Product.stock` is the canonical total base-unit count; `looseStock + Σ(packageCount × unitsPerPackage)` must equal `stock`
- Device name (`lan_device_name` in SharedPreferences) is used for sales attribution (`Sale.employeeId`) and receipt seller identification
- PIN protection guards sensitive operations (add/edit/delete products, create sales, edit receipts, apply discounts, etc.) via `PinProtection` helper and `PinService`; admin-forced flags (`pinForcedFlags` on `LicensePolicy`) are OR-ed with local prefs so admins can require PIN remotely
- Multi-device catalog sync uses LAN (`LanSyncService`) and Wi-Fi Direct (`WifiDirectSyncService`) only—no QR catalog sync; peers exchange full `SyncData` and product deletes propagate as tombstones (`deletedProductIds`, persisted via `ProductRepository`) applied on merge import; every sync service's `start()` checks `LicenseGate`
- On the sales page, multi-package products render as one catalog card per package (plus a Single line when applicable) and tap adds that line directly without a package picker sheet; cart-line quantity can also be typed as digits in the field between −/+ (commits on blur or keyboard done; stock-aware caps; 0 removes the line)
- Sales page tabs (Products/Cart/Receipts) support left/right swipe (`PageScrollPhysics` on `TabBarView`) plus a small bottom-right FAB that cycles tabs; the Receipts PIN gate is enforced on both swipe and tap via a `TabController` listener that reverts the index when the PIN is cancelled; product catalog page (`/products`) is titled "Inventory" in the AppBar and bottom-nav label (search hint "Search inventory…"), route path unchanged
- UI shares a design system: use `AppTokens` (e.g. `space2`/`space3`, `radiusM`, `border` width) and `AppThemeExtras` via `context.appExtras` (`panel`, `border`, `muted`, `danger`, `success`) for cards/panels/rows instead of hardcoded `Colors.white`/grey or literal radii, so styling adapts to dark mode and matches the global `cardTheme`
- Two-layer OTA updates: Shorebird (`shorebird.yaml` + `shorebird_code_push`, auto-checked via `ShorebirdUpdater` in `main.dart`) pushes Dart-only patches silently; `ApkUpdaterService` + `ApkUpdateWatcher` handle full-APK updates by fetching a JSON manifest (`update.json`, GitHub Releases by default), comparing `versionCode`, downloading to app storage, and launching the package installer — manifest URL overridable from Settings → Support (long-press), requires `REQUEST_INSTALL_PACKAGES` in AndroidManifest
- Admin/licensing stack lives in `lib/services/admin/` + `lib/ui/pages/admin/`: `install_id_service` registers each device, `device_heartbeat_service` + `usage_recorder` push to Firestore (`devicesCollection`, `adminsCollection`, `usageDaily` subcollection); `LicenseGate.ensure` guards sale/product writes and all sync `start()` calls, `LicenseWatcher` wraps the app and routes blocked devices to `/license-locked`; admins sign in via long-press on Settings → About (`/admin-login` → `/admin/*` shell), can set `LicensePolicy`, force PIN, and limit features; router uses `refreshListenable` on the policy stream; rules in `firestore.rules`, setup in `docs/admin_setup.md`
- Share App / download QR: `lib/config/app_share_config.dart` is the single source of `downloadUrl`/`appName`/`shareMessage`; `/share-app` (Settings → Support tile) renders a QR (`qr_flutter`) with native share via `share_plus` (falls back to copy on web/desktop); `web/index.html` shows a dismissible floating "Get ALEX on mobile" QR panel auto-hidden on ≤720px viewports — keep `AppShareConfig.downloadUrl` in sync with `data-download-url` in `web/index.html`
