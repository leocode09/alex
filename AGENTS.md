## Learned User Preferences

- Prefer auto-calculation of derived values over manual entry — if one number can be computed from others, compute it rather than asking the user to type it
- Design features for "full control": use flexible optional fields rather than rigid required ones (e.g. unit price is optional when every package has a fixed price)
- Bidirectional data entry: when A can derive B and B can derive A, support both directions so the user can start from whichever value they know
- When building an APK, copy the output file to the clipboard as a file (not just the path as text) so it can be pasted directly
- Prefer professional, theme-native UI — no gradients, rainbow accents, or decorative flourishes; use standard Material components, `ColorScheme` tokens, and flat/bordered panels

## Learned Workspace Facts

- Flutter POS/retail app using Riverpod for state, go_router for navigation, and JSON-based persistence via StorageHelper (not SQL); Firebase (Firestore + Auth) is used only for admin/licensing and cloud sync, never for local catalog/sales
- Product packages support per-package pricing (`packagePrice`), cost (`packageCostPrice`), inventory (`packageCount`), and loose stock (`looseStock`), with `Product.stock` as the canonical base-unit total (invariant: `looseStock + Σ(packageCount × unitsPerPackage) = stock`); `sellingPriceForPackage()` centralizes price resolution and `SaleItem.costPrice` captures per-base-unit cost at sale time (normalized from `packageCostPrice`), so profit displays on the catalog, receipts, receipt detail, and dashboard
- Device name (`lan_device_name` in SharedPreferences) is used for sales attribution (`Sale.employeeId`) and receipt seller identification
- PIN protection guards sensitive operations (add/edit/delete products, create sales, edit receipts, apply discounts, etc.) via `PinProtection` helper and `PinService`; admin-forced flags (`pinForcedFlags` on `LicensePolicy`) are OR-ed with local prefs so admins can require PIN remotely
- Multi-device data sync has three License-gated peers with the same surface (`start`/`stop`/`triggerSync`/`status`/`logs`): LAN (`LanSyncService`), Wi-Fi Direct (`WifiDirectSyncService`), and Firestore cloud backup in `lib/services/cloud/` (`CloudSyncService` + `ShopService` — anonymous auth + shop-code join, data partitioned under `/shops/{shopId}/…` per `FirestorePaths.syncedEntityCollections`, soft-deletes written as `{deleted: true}` on entities in `SoftDeletable`; `FirebaseInit.ensureInitialized()` never throws so missing Firebase config degrades to local-only); peers exchange full `SyncData`, product deletes propagate as tombstones (`deletedProductIds` via `ProductRepository`), and there is no QR catalog sync
- Sales page UX: tabs (Products/Cart/Receipts) support left/right swipe (`PageScrollPhysics` on `TabBarView`) plus a small bottom-right FAB that cycles tabs; the Receipts PIN gate is enforced on swipe and tap via a `TabController` listener that reverts the index when the PIN is cancelled; multi-package products render one catalog card per package (plus a Single line when applicable) and tap adds that line directly; cart-line quantity can be typed as digits in the field between −/+ (stock-aware caps; 0 removes); cart footer holds a `CustomerPickerSheet` tile (replacing the old free-text customer field) and the edit-receipt "Add Item to Receipt" dialog uses the same per-package row pattern via an `_AddEntry` holder; product catalog page (`/products`) is titled "Inventory" in the AppBar and bottom-nav label (search hint "Search inventory…"), route path unchanged; after Pay, `DataSyncTriggers.trigger` and auto-print are fire-and-forget (`unawaited` / background helper) so the checkout spinner is not blocked by sync peer timeouts or slow BLE receipt printing
- UI shares a design system: use `AppTokens` (e.g. `space2`/`space3`, `radiusM`, `border` width) and `AppThemeExtras` via `context.appExtras` (`panel`, `border`, `muted`, `danger`, `success`) for cards/panels/rows instead of hardcoded `Colors.white`/grey or literal radii, so styling adapts to dark mode and matches the global `cardTheme`
- Two-layer OTA updates: Shorebird (`shorebird.yaml` with a real `app_id`, bundled as a Flutter asset, `shorebird_code_push` auto-checked via `ShorebirdUpdater` in `main.dart`) pushes silent Dart-only patches — releases that should be patchable must come from `shorebird release android` then `shorebird patch android`; `ApkUpdaterService` + `ApkUpdateWatcher` handle full-APK updates by fetching a JSON manifest (`update.json`, default `https://github.com/leocode09/alex/releases/latest/download/update.json`), comparing `versionCode`, downloading to app storage, and launching the package installer — manifest URL overridable from Settings → Support (long-press), requires `REQUEST_INSTALL_PACKAGES` in AndroidManifest; release flow lives in `UPDATES.md` (bump `pubspec.yaml` version → `flutter build apk --release` → emit `update.json` with `versionCode`/`versionName`/`apkUrl`/`sha256` → `gh release create`)
- Admin/licensing stack lives in `lib/services/admin/` + `lib/ui/pages/admin/`: `install_id_service` registers each device, `device_heartbeat_service` + `usage_recorder` push to Firestore (`devicesCollection`, `adminsCollection`, `usageDaily` subcollection); `LicenseGate.ensure` guards sale/product writes and all sync `start()` calls, `LicenseWatcher` wraps the app and routes blocked devices to `/license-locked`; admins sign in via long-press on Settings → About (`/admin-login` → `/admin/*` shell), can set `LicensePolicy`, force PIN, and limit features; router uses `refreshListenable` on the policy stream; rules in `firestore.rules`, setup in `docs/admin_setup.md`
- Share App / download QR: `lib/config/app_share_config.dart` is the single source of `downloadUrl`/`appName`/`shareMessage`; `/share-app` (Settings → Support tile) renders a QR (`qr_flutter`) with native share via `share_plus` (falls back to copy on web/desktop); `web/index.html` shows a dismissible floating "Get ALEX on mobile" QR panel auto-hidden on ≤720px viewports — keep `AppShareConfig.downloadUrl` in sync with `data-download-url` in `web/index.html`
- Customer & rewards system: `Customer` carries `address`, `notes`, `creditBalance`, `totalBonusEarned`, `totalCreditRedeemed`; `Sale` carries `customerNameSnapshot`, `creditApplied`, `bonusEarned`, `customerTotalSpentAfter`, `customerCreditBalanceAfter` (sentinel-based `copyWith` so `null` can intentionally clear), plus `amountPaid` (defaults to `total`) with derived `amountDue` — checkout supports `'Pay Later'` (full total due) and partial cash (customer required when anything remains due); `SaleRepository.recordCustomerPayment` applies repayments FIFO across unpaid sales; `/customers/manage` aggregates due/credit/bonus-style stats (entry from customer list AppBar and Settings); profile shows outstanding sales and repayment; receipt preview + thermal print show Paid / Amount Due when not fully paid, and still include the Customer Rewards block; `CustomerCreditEntry` (bonus/redeem/manualAdjustment) is the ledger via `CustomerCreditRepository`; `BonusRule` (threshold/window/bonus) persists in SharedPreferences and is editable at `/settings/bonus-rule`; `BonusEngine` runs post-sale to grant bonuses (deduped by ledger lookup), record redemption lines, and stamp customer snapshots on the `Sale`; the checkout's "Apply credit" row is PIN-gated by the discount flag; ledger flows through `SyncData` / `CloudSyncService` / `SyncService` and the existing wildcard Firestore rule under `/shops/{shopId}` already covers `customerCreditEntries`
- Settings ("More") page is fully wired: Language opens a bottom-sheet picker backed by `lib/providers/language_provider.dart` (`AppLanguage` enum, persisted under `app_language`, English default with other locales flagged `comingSoon` but still saving the preference); Help Center is a real `/help` page (FAQ + contact via `share_plus` with copy fallback on desktop/web); About is a real `/about` page showing live `package_info_plus` version/build/`packageName`/Shorebird patch number with the admin-login long-press mirrored from the Settings tile onto the About logo; Bonus Rules editor lives at `/settings/bonus-rule`; the Reports page (`/reports`) has an AppBar `AppSearchField` toggle (sticky query across tab switches) plus a contextual `FilterChip` strip per tab (Sales: payment method/category; Inventory: category/stock state/variance reason; Sync: device/period — chips replaced the previous in-body dropdowns) with a Reset button shown only when a chip is active
